The table above shows the coefficients of the different attributes for linear regression. However, as the R-squared value was very low- 0.03, we can infer that this model does not accurately explain the variation in success.



From the sample of quantiles, we can see that the range of the different data points is spread across different values, and are congregated in 2 main clusters; clustering about 0 and 1. Furthermore, since the R- squared value is very low, we could possibly imply that the attributes may not have a linear relationship with the independent variable; i.e. the success of a terrorist. This is further supported by the residual plot below, whereby the residuals are not randomly distributed about the plot. There seems to be a pattern in the data. As such, we decided to use logistic regression for our data instead.

Logistic Regression - Success

From the accuracy results, we can see that the model is able to get 97% of the testing data predicted accurately. However, the accuracy of the null model is 96%. From the area under curve value, we can see the performance of the model is significantly high at 76% as well.

From the output above, we can see that the significant attributes are mainly the months of the attack. Based on the historical data, if the attacks are carried out in February it has the highest chance of success. Based on the results, if an attack occurs in February, the probability of the attack being successful increases by 41% as compared to other months. With this knowledge, special forces should try to hold their safety drills or runs in February instead. This will step up the security and may hinder terrorists from attacking. March also had a significantly large value compared to the other months. Thus, we can see that generally if the attacks are carried out in march or february is has a higher chance of succeeding as compared to the other months.


Logistic Regression - Multiple

We ran the logistic regression model for the multiple attacks as well. Our model achieved a 83% accuracy value. However, the null model is 83% as well. After doing the 10 fold cross validation, we can see that the average accuracy of the model is 83%.

Furthermore, the area under the curve value is 0.50. Thus this implies that the model is not really performing very well as it performs as well as the null model. In order to improve on the accuracy, we decided to build an ensemble learning method- Random Forests.
