Dataset

The data consists of 157,000 instances. The dataset was obtained from the Kaggle website. The raw data has a total of 137 different columns. During the preprocessing, we will analyse each column and see if it is necessary for the analysis. This data will be cleaned via data preprocessing and sanitisation.

Due to the large number of attributes, we have summarised in general the different types of attributes below :

s/n Description
1 Event ID
2 Date information (year, month, day)
3 Geographic information (country, state, city, longitude, latitude, etc.)
4 Attack type
5 Success/Failure
6 Type of Weapons Used
7 Hostage Situation

Data Preprocessing

In order to conduct a thorough and accurate analysis, we had to clean our data. We selected the attributes based on the whether these attributes could be obtained immediately after the first attack. As such, the final attributes were as follows.

s/n Description
1 Event ID
2 Date information (month and day)
3 Geographic information (specificity and region)
4 Attack type
5 Success/Failure
6 Type of Weapons Used
7 Number of Perpetrators
8 Property Damage
9 Extend of Property Damage
10 Multiple

We then removed all the rows with missing data points from the above attributes. We then converted some of the attributes which were of categorical nature into dummy variables to allow us to conduct the data analysis. For the association rules, we had to change the data into a binary format before the rules could be run. Thus, the dataset was duplicated and the data had to be formatted accordingly.
